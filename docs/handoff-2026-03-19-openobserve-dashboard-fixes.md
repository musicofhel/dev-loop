# Handoff — 2026-03-19 — OpenObserve Dashboard & Alert Fixes

## What was done

Ran a full verification plan (Phase 1–5) across the dev-loop test suite, calibration pipeline, and OpenObserve observability stack. All 863 tests pass. Calibration PASS with no regressions.

Found and fixed **4 classes of bugs** preventing dashboards and alerts from working:

### 1. Alert import delete bug (`scripts/import-alerts.py`)
- `delete_existing()` used `alert.get("id")` but OpenObserve v2 API returns `alert_id`
- Deletes silently no-op'd, stacking duplicates on every `--delete-existing` import
- **Fix**: `alert.get("id")` → `alert.get("alert_id")`

### 2. SQL type coercion errors (all alerts + all dashboards)
- `_timestamp` is `Int64` (epoch microseconds) in OpenObserve, not `Timestamp`
- `_timestamp >= NOW() - INTERVAL '10 MINUTES'` → type mismatch error
- `DATE_TRUNC('day', _timestamp)` → also fails (expects Timestamp input)
- **Fix**: `CAST(NOW() - INTERVAL '...' AS BIGINT) / 1000` for comparisons, `TO_TIMESTAMP(_timestamp / 1000000)` for DATE_TRUNC

### 3. Alert SQL bracket-accessor syntax
- Queries used `attributes_string['gate.status']` — invalid in OpenObserve
- OO flattens span attributes: dots→underscores (e.g. `gate.status` → `gate_status`)
- **Fix**: All queries now use flattened column names
- Also replaced `session_burn_rate` correlated subquery with static 200k threshold

### 4. Dashboard panel rendering failures
- `metric` type panels showed "Error Loading" or "No Data" for every chart type
- Root cause: OpenObserve dashboard panels with `customQuery: true` require time-series data (GROUP BY with time dimension). Scalar aggregate queries (`COUNT(*)` with no GROUP BY) never render.
- Also: empty `x`/`y` field arrays in panel config caused silent failures
- **Fix**: (a) All former `metric` panels converted to time-bucketed `area`/`bar` charts with `GROUP BY day/hour`. (b) Import script auto-generates `x`/`y` field definitions from SQL SELECT aliases.

### 5. Misc import script fixes
- `import-dashboards.py`: `v8` can be `None` for UI-created dashboards → null guard added
- `calibration.json`: `span_name` → `operation_name` (OO's actual field name)
