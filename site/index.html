How It Works

AI coding agents are fast — but they can delete your .env, leak API keys, run rm -rf /, or push broken code. The faster they work, the more damage they can do before you notice.

dev-loop sits between the AI agent and your codebase. It intercepts every action, checks it against safety rules, and blocks anything dangerous — all before the action executes.

Three Tiers of Protection

Tier 1 — Real-time

< 5ms per check

Every file write, edit, or shell command is checked against deny lists, dangerous patterns, and secret detectors before it executes.

Tier 2 — Commit-time

~30 seconds

Tests, security scanning (Semgrep), secret detection, and spec enforcement run in sequence before each commit goes through.

Tier 3 — Full pipeline

On demand

Take a bug report, assign it to an agent, agent writes code, quality gates run, retry on failure, PR created. The full closed loop.

What Gets Blocked

The AI tries to... What happens
Write to .env or .ssh/config Blocked before the write executes (< 5ms)
Run rm -rf / or curl ... | sh Blocked before the command executes (< 5ms)
Force-push to main Blocked before the push executes (< 5ms)
Commit code containing an API key Blocked at commit time (~30s)
Commit code with a known vulnerability Blocked at commit time (Semgrep scan)
Commit code that fails tests Blocked at commit time (auto-detected test runner)

Check Engine

Deny List — 17 patterns

Blocks writes to sensitive files: .env, .ssh/*, *.key, *.pem, credentials, keystores.

Dangerous Ops — 25 patterns

Catches risky commands: rm -rf, force push, DROP TABLE, curl | sh, chmod 777, sudo rm.

Secret Scanner — 15 patterns

Detects API keys, tokens, private keys, database connection strings, and high-entropy secrets in file content.

Seven-Layer Architecture

Seven layers form a closed loop — the output of every stage feeds back as input. There is no "end", only cycles that get tighter as the harness learns.

# Layer What it does
1 Intake Pulls issues from the tracker (beads), resolves dependencies, assigns priority
2 Orchestration Creates isolated git worktrees, selects agent personas, decomposes tasks
3 Agent Runtime Runs the AI agent with scoped permissions, CLAUDE.md overlays, MCP tool access
4 Quality Gates Tests, Semgrep security scans, secret detection, spec enforcement, cost limits
5 Observability OpenTelemetry spans, OpenObserve dashboards, NDJSON session replay
6 Feedback Loop Retries failures, escalates to humans, detects patterns, triggers cross-repo cascades
7 LLMOps DSPy/GEPA prompt optimization using real session data (feature-flagged)

Quick Start

# Build and install the daemon
cd daemon && cargo build --release
cp target/release/dl ~/.local/bin/dl

# Hook into Claude Code
dl install

# Start the daemon
dl start

# Check status
dl status

Once running, dev-loop silently protects every Claude Code session. No changes to your workflow needed.
