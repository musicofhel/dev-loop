## Phase 1: Fix Import Script Root Causes

*These bugs in `scripts/import-dashboards.py` cascade across multiple dashboards.*

### 1a. Fix `_make_fields()` x-axis auto-detection (line ~110-180)

The heuristic places computed columns (e.g. `failure_pct`, `avg_lead_time_s`, `block_pct`, `avg_recovery_s`) into `fields.x` alongside the time dimension. OO then interleaves metric values with date labels on the x-axis.

**Visible symptoms:**
- DORA Panel 3: garbled x-axis (failure_pct values interspersed with dates)
- DORA Panels 2, 4: latent (avg_lead_time_s / avg_recovery_s in fields.x)
- Calibration Panels 2, 6: block_pct in both x and y
- Quality Gates Panel 5: gate and day conflated on x-axis

**Fix:** In `_make_fields()`, the condition `not is_agg and has_group_by` incorrectly classifies computed CASE/ROUND expressions as non-aggregates. Tighten the detection: only raw GROUP BY dimension columns should go to x, all computed/derived columns go to y.

### 1b. Remove `"area": "area-stacked"` from `PANEL_TYPE_MAP` (line 32)

OO lacks plain `area`, but this silent mutation misleads config readers. Either: keep the mapping but add a comment explaining why, or find if OO now supports `area`.

**Affected:** 10 panels across 4 dashboards (Agent Perf 1-3, Loop Health 2/3/6, DORA 2/4, Cost Tracking 1/4)

### 1c. Fix `_fix_aggregate_timestamp()` mutation (line 183-203)

Prepends `MIN(_timestamp) as _timestamp` to pure-aggregate queries, which changes semantics for detail-listing queries (e.g. Cost Tracking Panel 5's top-N query has no GROUP BY, so adding MIN() creates a conflict).

**Fix:** Only apply when the query truly needs it. Skip queries with ORDER BY + LIMIT that are clearly detail listings.

## Phase 2: Fix Hardcoded SQL Time Intervals

*Universal defect — all 34 panels across all 6 dashboards hardcode `INTERVAL 'N DAYS'`.*

**Impact:** The dashboard time picker is completely non-functional for every panel.

**Fix:** Replace `_timestamp >= CAST(NOW() - INTERVAL '30 DAYS' AS BIGINT) / 1000` with OO's time range injection mechanism. Investigate whether OO supports `$__timeFilter` or equivalent. If not, remove the hardcoded WHERE clause entirely and let OO's time picker inject the filter.

## Phase 3: Fix Agent Performance Dashboard

*6 panels, all render data but with multiple quality issues.*

### 3a. [Critical] Fix persona scope bug

Panels 4 & 5 use `operation_name LIKE 'tb%.phase.persona'` but only query `tb1_persona`, missing TB2-TB6 data.

**Fix:** Use `COALESCE(tb1_persona, tb2_persona, tb3_persona, tb4_persona, tb5_persona, tb6_persona)` or narrow the operation_name filter to match only TB1.

### 3b. [Critical] Fix duration divisor

Panel 4: `duration / 1000000` labeled "Avg Seconds" — OTel durations are nanoseconds, so this yields milliseconds, not seconds. Divisor should be `1000000000` for seconds.

### 3c. [Critical] Remove redundant panels 2 & 3

Panels 2 (Avg Input Tokens) and 3 (Avg Output Tokens) duplicate Panel 6 (Token Usage Over Time). Remove them and promote Panel 6 to position 2. This frees above-fold space.

### 3d. [Warning] Fix Panel 6 WHERE clause

Add `AND runtime_output_tokens IS NOT NULL` — currently only checks input tokens.

### 3e. [Warning] Fix color inconsistency

Input tokens: salmon in Panel 2 vs yellow in Panel 6. Output tokens: green in Panel 3 vs magenta in Panel 6. Assign consistent colors per metric.

### 3f. [Minor] Fix "Per Day" title mismatch

Panels 1-3 say "Per Day" but x-axis shows sub-day (6-hour) granularity. Rename to "Over Time" or aggregate to true daily buckets.

## Phase 4: Fix DORA Metrics Dashboard

*4 panels, 1 dead, 2 duplicates, 1 garbled.*

### 4a. [Critical] Fix Panel 1 dead operation name

`operation_name = 'intake.issue_pickup'` doesn't exist in trace data. Change to an existing span name (e.g. `tb%.run` with `COUNT(DISTINCT trace_id)` — similar to Loop Health Panel 1).

### 4b. [Critical] Deduplicate Panels 2 & 4

Both compute `AVG(duration / 1000000)` for `operation_name LIKE 'tb%.run'`. Panel 4 ("Recovery Time Trend") should measure actual recovery — e.g., time between `gate_status = 'fail'` and next `gate_status = 'pass'` on the same trace, or filter for retry-specific spans.

### 4c. [Critical] Garbled x-axis on Panel 3

Already fixed by Phase 1a (`_make_fields()` fix). After reimport, `failure_pct` will be in y-axis only.

### 4d. [Minor] Fix y-axis labels

"Avg Lead Time S" → "Avg Lead Time (s)", "Failure Pct" → "Failure Rate (%)", "Avg Recovery S" → "Recovery Time (s)"

## Phase 5: Fix Loop Health Dashboard

*6 panels, all render data but with data quality concerns.*

### 5a. [Critical] Fix NULL vs empty string ambiguity

Panels 2 & 4 use `gates_first_failure IS NULL` / `IS NOT NULL`. OO may store absent Utf8 values as empty strings.

**Fix:** `(gates_first_failure IS NULL OR gates_first_failure = '')` for Panel 2, `(gates_first_failure IS NOT NULL AND gates_first_failure != '')` for Panel 4.

### 5b. [Warning] Add test span exclusion

All 6 panels should add `AND (devloop_test IS NULL OR devloop_test != 'true')` to prevent test span contamination.

### 5c. [Warning] Validate retry rate metric (Panel 6)

Day 12 peaks near 100%. Investigate: are `feedback.retry` spans generating their own `tb%.run` spans (inflating denominator)? Can one run generate multiple retries (inflating numerator)?

### 5d. [Minor] Humanize gate names (Panel 4)

X-axis shows `gate_0_sanity`, `gate_3_security`, etc. Use SQL `CASE` or a lookup to show "Sanity Check", "Security Gate", "Dangerous Ops Guard".

## Phase 6: Fix Quality Gate Insights Dashboard

*5 panels, 3 render data but with critical config issues.*

### 6a. [Critical] Add "passed" series to Panel 1

Query computes both `passed` and `failed` but only `failed` is in y-axis fields. Add `passed` to y-axis with green color (`#4CAF50`). Or rename panel from "Gate Pass/Fail Rates" to "Gate Failure Counts".

### 6b. [Critical] Fix Panel 5 x-axis conflation

`gate` (operation_name) is in `fields.x` alongside `day`. Move `gate` to a breakdown/series field so each gate becomes a separate line. (Partially addressed by Phase 1a, but may need manual config adjustment in source JSON.)

### 6c. [Critical] Fix CWE duplication (Panel 3)

`security_cwe_ids` stores "CWE-89,CWE-89" as a raw string.

**Fix:** Use `UNNEST(STRING_SPLIT(security_cwe_ids, ','))` to split and deduplicate, or fix source instrumentation.

### 6d. [Warning] Utf8 numeric cast safety

`gate_duration_ms` and `gate_findings_count` are Utf8. Add regex guards: `AND gate_duration_ms ~ '^[0-9]'`

### 6e. [Minor] Fix x-axis label truncation (Panels 1-2)

Long `operation_name` values get garbled with digit-overlap rendering. Consider horizontal bar layout or shorter aliases in SQL.

## Phase 8: Cross-Cutting Polish

*Lower priority improvements that affect multiple dashboards.*

### 8a. Color palette design

All dashboards use `#5960b2` as default. OO overrides it inconsistently. Design an intentional color palette:
- Green for success/pass metrics
- Red/salmon for failure/retry metrics
- Blue for neutral time-series
- Distinct colors per metric across panels

### 8b. Add KPI summary rows

No dashboard has at-a-glance stat panels. Add a top row of single-number panels:
- Agent Perf: total runs, total tokens, avg duration, active personas
- Loop Health: total issues, success rate, avg lead time, retry rate
- Quality Gates: total runs, pass rate, total findings, mean duration
- DORA: runs today, avg lead time, failure rate, recovery time

### 8c. Fix y-axis decimal formatting on COUNT metrics

Loop Health Panels 1/4/5 and Quality Gates Panels 1-2 show "210.00", "35.00" on integer counts. Format as integers.

### 8d. Add month context to x-axis dates

All time-series panels show "14, 15" without month. Show "Mar 14" format.
